# AID web frontend — walkthrough & air-gapped evidence

The GUI is MoonBit→JS (`ui/src`, compiled to `ui/static/app.js`) over the REST API,
with Bootstrap 5 bundled locally (no CDN). Everything is served by `aid serve` from
the embedded `ui/static/`. It is a **first-class topology designer** (not just a
viewer): create/edit/duplicate/delete plans, a structured server/switch/connection
editor with a mesh↔Clos selector, live (no-persist) validation as you design, an
optic-overlay tab, at-a-glance derived facts, and per-fabric wiring + BOM-CSV export.
The engine is the rebuilt `internal/design` path (F7b): the calc `CalcOutput`
(`is_valid` + switch/server quantity arrays + endpoints + transceiver verdicts) and
the flat BOM `rows[]`.

## Reproduce (live)

```bash
# 1. Build the bundle + the single static binary (embeds ui/static via go:embed).
make ui
go build -o aid ./cmd/aid

# 2. Run the server with an empty plans dir.
./aid serve --port 8080 --plans-dir /tmp/aid-plans

# 3. Seed a plan from a DIET/training bundle, then attach its optic overlay (the
#    companion sub-resource). Any committed XOC composition works.
curl -s -X POST --data-binary @tests/oracle/xoc-256-2xopg128-clos-ro/training.yaml \
  http://localhost:8080/api/plans                                   # -> {"id": "<id>", ...}
curl -s -X PUT  --data-binary @tests/oracle/xoc-256-2xopg128-clos-ro/optic-overlay.yaml \
  http://localhost:8080/api/plans/<id>/overlay

# 4. Open http://localhost:8080/ and walk: plan list -> New plan / View ->
#    structured-edit + live-validate -> Calculate -> View BOM -> Download wiring.
```

### Air-gapped evidence (regenerated by driving the real API)

The evidence under `ui/docs/evidence/` and `ui/docs/screenshots/` is **generated by
driving the real REST API + GUI in headless chromium against a live seeded
`aid serve`** — every page is captured *after* a real request round-trip (the
seeder POSTs the plans and PUTs the overlays; calc / BOM / wiring / facts / live
validation are computed by the live engine). It therefore doubles as an end-to-end
smoke test, and the generator **fails if the page issues any non-same-origin
request** (the air-gapped / no-CDN guarantee).

```bash
make ui-evidence   # boots a seeded server and refreshes normalized HTML evidence
AID_UPDATE_SCREENSHOTS=1 make ui-evidence   # also refreshes PNG screenshots
```

PNG screenshots are opt-in because headless-browser image output varies across
hosts and browser revisions. The default evidence target updates the deterministic
HTML/request artifacts and leaves the tracked screenshot snapshot unchanged.

The HTML pages are self-contained (they link the locally-bundled Bootstrap CSS) and
open offline. The captured request manifest is written to
[`evidence/requests.txt`](evidence/requests.txt). Same offline shims as `make ui-e2e`
(chromium via `PLAYWRIGHT_BROWSERS_PATH`, `@playwright/test` via an ephemeral
`ui/test/node_modules` symlink); CI must provide both — see `ui/test/README.md`.

## Walkthrough

### 1. Plan list (`GET /api/plans`)
A NetBox-style dark table of plans with status badges, a per-row derived-facts
summary (topology / GPU / server / switch totals + a Valid/Invalid cue), and per-row
**View / Duplicate / Delete**. Rows are clickable to open detail; a first-run
empty-state offers New-from-template. Screenshot: `screenshots/01-plan-list.png`.
[`evidence/01-plan-list.html`](evidence/01-plan-list.html)

### 2. New plan — authoring (`GET /api/templates`, `POST /api/plans`)
A name field, a starter-template picker, and the raw-YAML editor. From-template
prefills the YAML and attaches the optic overlay on create (so the BOM optics are
populated). Screenshot: `screenshots/02-new-plan.png`.
[`evidence/02-new-plan.html`](evidence/02-new-plan.html)

### 3. Plan detail (`GET /api/plans/{id}`, `…/structure`, `…/overlay`)
The derived-facts header; the action toolbar with one clear primary (**Calculate**);
the **structured editor** (server/switch/connection forms with dropdowns sourced
from the plan's catalog + an explicit mesh|clos selector); the raw-YAML escape
hatch; and the optic-overlay tab. Screenshot: `screenshots/03-plan-detail.png`.
[`evidence/03-plan-detail.html`](evidence/03-plan-detail.html) ·
[`evidence/07-overlay.html`](evidence/07-overlay.html)

### 4. Live validation as you design (`POST /api/validate`)
Editing the plan (raw YAML or any structured field) fires a debounced dry-run that
validates **without persisting** — the inline result shows the live Valid/Invalid
verdict and computed quantities, so over-allocation is caught before save.
[`evidence/08-live-validation.html`](evidence/08-live-validation.html)

### 5. Calculate (`POST /api/plans/{id}/calc`) — two-plane validation
A green **Valid** cue + the computed **switch/server quantities** (per class), an
endpoint/verdict summary, and a **Download wiring** button per managed fabric
(hhfab CRDs). A calc constraint violation renders a red **Invalid** cue with the
error codes (e.g. `ZONE_OVERFLOW`), surfaced as data (HTTP 200) — *not* an error
alert; no wiring is offered for an invalid calc. (A *structural* failure — a 4xx
`{"error": ...}` body — would render a distinct danger alert, never a validity
cue.) Screenshot: `screenshots/04-calc.png`.
[`evidence/04-calc-valid.html`](evidence/04-calc-valid.html) ·
[`evidence/05-calc-invalid.html`](evidence/05-calc-invalid.html)

### 6. BOM (`GET /api/plans/{id}/bom`)
A flat line-item table (section / model / hedgehog class / manufacturer / quantity +
optic standard) with the suppressed-cable-assembly footer and a **Download BOM
(CSV)** export. Optic columns are populated when an overlay is attached. Screenshot:
`screenshots/05-bom.png`.
[`evidence/06-bom.html`](evidence/06-bom.html)

## Air-gapped run (no network; all assets from the binary)

Every request the page makes is same-origin (served from the embedded FS) — there
are **no CDN / external requests**. The generator records the live manifest and
fails on any external request; the captured set (see
[`evidence/requests.txt`](evidence/requests.txt)) is:

```
/                                  /api/plans/<id>/calc
/static/app.js                     /api/plans/<id>/bom
/static/bootstrap.min.css          /api/plans/<id>/overlay
/static/bootstrap.bundle.min.js    /api/plans/<id>/structure
/api/plans                         /api/templates
/api/plans/<id>                    /api/validate

external (non-same-origin) requests: 0
PASS: all assets loaded from the binary (no CDN).
```
